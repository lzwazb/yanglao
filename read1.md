# 项目进展总结

项目目前是一个基于前后端分离架构的养老服务管理系统。
- **后端**：采用 Java Spring Boot 微服务架构，拆分为：
    -   `administrator-server`（管理员服务）
    -   `login-server`（登录与注册服务）
    -   `user-server`（用户信息服务）
    -   `business-server`（业务中心服务，负责生活服务等核心业务）
- **前端**：采用 Vue.js 框架，并使用 Vite 进行构建。

**已完成的核心功能**：
1.  **用户体系**：实现了用户、家属、管理员三种角色的注册和登录功能。
2.  **管理后台**：
    -   管理员可以对系统内的所有用户（包括家属、普通用户）进行增、删、改、查等基本管理。
    -   **服务管理**：管理员可以发布、编辑、上下架服务项目，并对用户的服务订单进行接单、完成或取消操作。
3.  **个人中心**：普通用户和家属可以查看和修改自己的个人资料。
4.  **生活服务模块**：
    -   **服务展示**：用户可浏览各类养老服务项目（如家政、助餐等）。
    -   **在线预约**：用户可提交“登记入住”或“上门服务”的预约申请。
    -   **订单管理**：用户可查看自己的预约状态，并取消未确认的订单。
    -   **服务评价**：用户可对已完成的服务订单进行评分和评价。

# 未完成的功能

根据前端 `src/views` 目录下已创建但尚未完全实现的页面组件，可以识别出以下主要功能模块待开发：

1.  **活动管理** (`activity` 目录):
    -   **功能**：发布、展示、报名和管理养老中心的各类活动。
    -   **前端**：已创建 `ActivityList.vue` 和 `ActivityManage.vue` 页面。
    -   **后端**：需要相应的微服务来支持活动信息的增删改查及报名逻辑（建议集成在 `business-server` 中）。

2.  **健康管理** (`health` 目录):
    -   **功能**：记录和追踪老年人的健康数据（如体征信息、用药提醒、体检报告等）。
    -   **前端**：已创建 `HealthManagement.vue` 页面。
    -   **后端**：需要设计健康档案相关的数据库表和API接口（建议新建 `health-server`）。

3.  **留言板** (`message` 目录):
    -   **功能**：提供一个用户之间或用户与管理员之间交流的平台。
    -   **前端**：已创建 `MessageBoard.vue` 页面。
    -   **后端**：需要实现留言的发布、查看、回复和管理功能。

4.  **公告通知** (`notice` 目录):
    -   **功能**：向所有用户发布重要的通知或公告。
    -   **前端**：已创建 `NoticeList.vue` 页面。
    -   **后端**：需要支持公告的发布、置顶和历史查询。
