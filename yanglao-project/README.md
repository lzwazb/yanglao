# 养老服务平台前端项目

基于 Vue3 + Vite + Element Plus 开发的养老服务平台前端应用。

## 功能模块

### 1. 注册登录模块
- 用户登录（老人、家人、管理员、员工）
- 用户注册（老人注册、家人注册）

### 2. 信息管理（管理员）
- 用户管理（老人）
- 家人管理
- 员工管理
- 管理员管理

### 3. 服务预约
- 登记入住（房间号、床位号、服务编号）
- 上门服务（单元号、房间号、服务编号）
- 生活服务（家政保洁、助餐助浴等）

### 4. 服务评价
- 用户对服务的评价和评分

### 5. 健康管理
- 健康数据记录
- 健康检测预约
- 健康管家服务

### 6. 活动管理
- 活动列表展示
- 活动报名
- 活动管理（管理员）

### 7. 公告栏
- 社区公告展示
- 顶部滚动通知

### 8. 留言板
- 日常交流
- 社区建议

### 9. 个人中心
- 个人信息修改
- 查看报名活动
- 查看健康数据
- 修改密码

### 10. 家人功能
- 我的长辈（查看关联老人的信息、健康数据、服务记录、活动记录）

## 技术栈

- Vue 3
- Vite
- Vue Router
- Pinia
- Element Plus
- Axios

## 安装依赖

```bash
npm install
```

## 开发运行

```bash
npm run dev
```

## 构建生产版本

```bash
npm run build
```

## 后端接口配置

项目已配置代理，后端服务地址：
- 登录/注册服务：http://localhost:5000
- 管理员服务：http://localhost:5001

代理配置在 `vite.config.js` 中，可根据实际情况修改。

## 注意事项

1. 部分功能模块的后端接口可能还未完全实现，前端已预留接口调用位置，待后端接口完成后可补充完整。
2. 需要确保后端服务已启动并正常运行。
3. 登录后，用户信息会存储在 localStorage 中。

## 开发说明

- 使用 Element Plus 作为 UI 组件库
- 使用 Pinia 进行状态管理
- 使用 Axios 进行 HTTP 请求
- 路由守卫已配置，未登录用户会自动跳转到登录页
- 不同角色用户会看到不同的菜单项
